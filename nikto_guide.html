Kali Linux Security Tools

NIKTO

คู่มือการใช้งาน Web Server Vulnerability Scanner
ฉบับภาษาไทย พร้อมตัวอย่างและคำอธิบายผลลัพธ์

ตรวจสอบได้กว่า 6,700+ รายการช่องโหว่  ·  Open Source  ·  Perl-based
คำเตือนสำคัญก่อนใช้งาน

Nikto ใช้สแกนหาช่องโหว่ของเว็บเซิร์ฟเวอร์ได้เฉพาะกับเว็บที่คุณเป็นเจ้าของ หรือได้รับอนุญาตเป็นลายลักษณ์อักษรเท่านั้น เช่น แล็บส่วนตัว, DVWA, Metasploitable, TryHackMe, HackTheBox

การสแกนเว็บไซต์ของผู้อื่นโดยไม่ได้รับอนุญาตถือเป็นความผิดทางกฎหมายในหลายประเทศ รวมถึงไทยตาม พ.ร.บ.คอมพิวเตอร์ แม้จะ "แค่สแกน" ก็ตาม ตัวอย่าง IP/URL ในคู่มือนี้เป็นของแล็บทดสอบส่วนตัวเท่านั้น

01

Nikto คืออะไร

Nikto เป็น open-source web server scanner เขียนด้วย Perl ใช้ตรวจสอบเว็บเซิร์ฟเวอร์หาปัญหาด้านความปลอดภัยแบบอัตโนมัติ

ไฟล์/โปรแกรมอันตรายหรือล้าสมัยบนเซิร์ฟเวอร์
เวอร์ชันซอฟต์แวร์เก่าที่มีช่องโหว่ CVE ที่รู้จัก
การตั้งค่า HTTP header ที่ไม่ปลอดภัย
ไฟล์ default/config ที่ไม่ควรเปิดเผย เช่น .htaccess, backup
Directory Indexing — ไดเรกทอรีที่เปิดให้ list ได้
Cookie ที่ไม่ตั้งค่า secure/httponly และ XSS protection header
ติดตั้งและเริ่มต้นใช้งาน
ติดตั้งบน Kali Linux (ปกติมีอยู่แล้ว)
sudo apt update && sudo apt install nikto -y
ตรวจสอบเวอร์ชัน
nikto -Version
อัปเดตฐานข้อมูล plugin/signature ก่อนใช้งาน
nikto -update
02

Syntax พื้นฐาน

รูปแบบหลัก
nikto -h <target> [OPTIONS]
รูปแบบที่ใช้บ่อย
nikto -h <IP_หรือ_URL>
nikto -h <IP> -p <PORT>
nikto -h <URL> -ssl
nikto -h <IP> -o <output_file> -F <format>
03

คำอธิบาย Flag ทั้งหมด

กลุ่มเป้าหมาย
-h, -host ระบุเป้าหมาย (IP, hostname, หรือ URL เต็ม) เช่น -h 10.10.10.5 หรือ -h http://10.10.10.5
-p, -port ระบุพอร์ต (default 80, หรือ 443 ถ้าใช้ -ssl) รองรับหลายพอร์ต เช่น -p 80,443,8080
-ssl บังคับใช้ SSL/TLS (https) ในการเชื่อมต่อ
-root กำหนด base path ที่จะสแกน กรณีเว็บอยู่ใน subdirectory เช่น -root /dvwa/
กลุ่มควบคุมการสแกน
-Tuning เลือกประเภทการทดสอบเฉพาะ (ดูตารางด้านล่าง)
1Interesting File / Seen in logs
2Misconfiguration / Default File
3Information Disclosure
4Injection (XSS/Script/HTML)
5Remote File Retrieval - Inside Web Root
6Denial of Service
7Remote File Retrieval - Server Wide
8Command Execution / Remote Shell
9SQL Injection
0File Upload
xReverse Tuning (ใช้ exclude แทน)
-evasion เลือกเทคนิคหลบ IDS/IPS (1–8) เช่น URL encoding, directory self-reference เพื่อเลี่ยงการตรวจจับ
-timeout เวลา timeout ต่อ request (default 10 วินาที)
-maxtime จำกัดเวลาสแกนทั้งหมด เช่น -maxtime 1h
