Kali Linux · Penetration Testing Tool

gobuster_

คู่มือ Directory / File / DNS / Vhost Brute-force ฉบับสมบูรณ์

⚠️
⚠ คำเตือนสำคัญก่อนใช้งาน

Gobuster ใช้ได้กับเป้าหมายที่คุณเป็นเจ้าของ หรือได้รับอนุญาตเป็นลายลักษณ์อักษรเท่านั้น เช่น แล็บส่วนตัว, DVWA, Metasploitable, TryHackMe, HackTheBox — การสแกนเป้าหมายของผู้อื่นโดยไม่ได้รับอนุญาต ผิดกฎหมาย ตัวอย่าง IP/URL ในคู่มือนี้เป็นของแล็บทดสอบส่วนตัวเท่านั้น

01 — Overview

Gobuster คืออะไร

Gobuster เป็นเครื่องมือเขียนด้วยภาษา Go ใช้ brute-force หาสิ่งที่ "ซ่อนอยู่" บนเว็บไซต์หรือระบบ DNS โดยอาศัย wordlist ยิงเข้าไปทีละคำแล้วดู response จุดเด่นคือเร็วมาก เพราะ compile เป็น binary และรองรับ multi-thread

dir
หา directory/file ที่ซ่อนอยู่บนเว็บ (ใช้บ่อยที่สุด)
dns
หา subdomain ของโดเมนเป้าหมาย
vhost
หา virtual host บนเซิร์ฟเวอร์เดียวกัน
fuzz
Fuzzing ทั่วไปด้วย FUZZ keyword
s3
หา AWS S3 bucket
gcs
หา Google Cloud Storage bucket
tftp
Brute-force TFTP filename
bash — ติดตั้ง
sudo apt update && sudo apt install gobuster -y
gobuster version
02 — Syntax

Syntax พื้นฐาน

รูปแบบทั่วไป
gobuster <mode> [OPTIONS]

gobuster dir  -u <URL>    -w <wordlist> [OPTIONS]
gobuster dns  -d <domain> -w <wordlist> [OPTIONS]
gobuster vhost -u <URL>   -w <wordlist> [OPTIONS]
gobuster fuzz -u <URL_with_FUZZ> -w <wordlist> [OPTIONS]
03 — Flags

คำอธิบาย Flag ทั้งหมด

Flagคำอธิบาย
Global (ใช้ได้ทุกโหมด)
-w, --wordlistไฟล์ wordlist ที่ใช้ brute-force (จำเป็นเสมอ)
-w /usr/share/wordlists/dirb/common.txt
-t, --threadsจำนวน thread ที่รันพร้อมกัน (default 10) ยิ่งสูงยิ่งเร็ว แต่เสี่ยงโดนบล็อก
-t 50
-o, --outputบันทึกผลลัพธ์ลงไฟล์
-o result.txt
-v, --verboseแสดงผลแบบละเอียด (รวมผลที่ "ไม่เจอ" ด้วย)
-q, --quietซ่อน banner แสดงแต่ผลลัพธ์
-z, --no-progressไม่แสดง progress bar (เหมาะกับการ pipe output)
--delayหน่วงเวลาระหว่าง request ใช้เลี่ยง rate-limit/WAF
--delay 100ms
dir — หา Directory/File
-u, --urlเป้าหมาย
-u http://10.10.10.5
-x, --extensionsนามสกุลไฟล์ที่จะลองเติมต่อท้ายคำ ทำให้ได้ผลชัดขึ้น
-x php,html,txt,bak
-s, --status-codesแสดงเฉพาะ HTTP status code ที่ต้องการ
-s 200,301,403
-b, --status-codes-blacklistซ่อน status code ที่ระบุ (ตรงข้ามกับ -s)
-b 404
-e, --expandedแสดง URL เต็มรูปแบบ (http://...) แทน path สั้น
-f, --add-slashเติม "/" ท้าย request เช็คว่าเป็น directory จริงหรือไม่
-r, --follow-redirectติดตาม HTTP redirect (3xx) อัตโนมัติ
-k, --no-tls-validationข้ามการตรวจสอบ SSL cert (กรณีใช้ self-signed cert ในแล็บ)
-H, --headersเพิ่ม HTTP header custom
-H "Authorization: Bearer xxx"
-c, --cookiesส่ง cookie ไปด้วย
-c "PHPSESSID=abc123"
--username / --passwordใช้ HTTP Basic Auth
-m, --methodเลือก HTTP method (default GET)
-m POST
dns — หา Subdomain
-d, --domainโดเมนเป้าหมาย
-d example.lab
-i, --show-ipsแสดง IP address ของ subdomain ที่เจอด้วย
-c, --show-cnameแสดง CNAME record (ถ้ามี)
-r, --resolverระบุ DNS resolver ที่ใช้ query
-r 8.8.8.8
vhost — หา Virtual Host
-u, --urlIP/URL ของเซิร์ฟเวอร์
--domainโดเมนหลักที่ใช้สร้าง vhost ทดสอบ
--append-domainเติมโดเมนต่อท้ายคำใน wordlist อัตโนมัติ
04 — Examples

ตัวอย่างการใช้งาน

4.1หา Directory/File พื้นฐาน (dir)
สถานการณ์: เว็บแล็บ DVWA บน IP 10.10.10.5 ต้องการหา directory/file ที่ซ่อนอยู่
bash
gobuster dir \
  -u http://10.10.10.5 \
  -w /usr/share/wordlists/dirb/common.txt \
  -t 50 \
  -x php,html,bak,txt \
  -o gobuster_result.txt
