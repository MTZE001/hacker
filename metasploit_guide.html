Metasploit Framework

msfconsole · Exploitation Framework บน Kali Linux

⚠ คำเตือนสำคัญก่อนใช้งาน

Metasploit ใช้สำหรับ เจาะระบบจริง (exploit) เพื่อเข้าควบคุมเครื่องเป้าหมาย ต่างจากเครื่องมือสแกน/เดารหัสที่ผ่านมา ความเสี่ยงทางกฎหมายสูงกว่ามาก ใช้ได้เฉพาะกับ:

  • แล็บที่สร้างมาเพื่อฝึกฝนโดยเฉพาะ เช่น Metasploitable2/3, TryHackMe, HackTheBox, VulnHub
  • ระบบที่คุณเป็นเจ้าของ หรือมีสัญญา/เอกสารอนุญาต pentest เป็นลายลักษณ์อักษรชัดเจน (scope of work)

การยิง exploit เข้าระบบของผู้อื่นโดยไม่ได้รับอนุญาต ไม่ว่าจะสำเร็จหรือไม่ ถือเป็นความผิดทางกฎหมายร้ายแรง

01 METASPLOIT คืออะไร

Metasploit Framework (MSF) เป็นชุดเครื่องมือสำหรับพัฒนา ทดสอบ และรัน exploit code แบบเป็นระบบ มี database exploit/payload ที่อัปเดตสม่ำเสมอ และมี interface หลักชื่อ msfconsole

MODULE TYPES
exploit

โค้ดที่ใช้ "เจาะ" ช่องโหว่เพื่อเข้าระบบ

payload

โค้ดที่รันหลัง exploit สำเร็จ เช่น เปิด shell

auxiliary

โค้ดเสริม เช่น scanner, fuzzer, DoS ช่วยเก็บข้อมูล

post

โค้ดที่รันหลังเข้าระบบได้แล้ว เช่น privilege escalation

encoder

ใช้เข้ารหัส payload เพื่อหลีกเลี่ยงตัวอักษรที่เป็นปัญหา

nop

Generator no-operation code ใช้ใน buffer overflow

SESSION

การเชื่อมต่อที่เปิดอยู่หลัง exploit สำเร็จ เช่น meterpreter session, shell session ใช้ ID อ้างอิงเมื่อต้องจัดการหลาย session

ติดตั้งและเริ่มต้นใช้งาน
# ติดตั้ง (Kali ปกติมีอยู่แล้ว)
sudo apt update && sudo apt install metasploit-framework -y
# ครั้งแรก: สร้าง database ก่อน
sudo msfdb init
msfconsole
02 คำสั่งพื้นฐานใน msfconsole
search <keyword>ค้นหา module จาก keyword / CVE / ชื่อ
use <module_path>เลือก module มาใช้งาน
infoแสดงรายละเอียดของ module ที่เลือกอยู่
show optionsแสดงค่า parameter ที่ต้องตั้งของ module
show payloadsแสดง payload ที่ใช้ได้กับ exploit นี้
show targetsแสดงเป้าหมาย (OS/version) ที่ module รองรับ
set <OPTION> <VALUE>ตั้งค่า parameter เช่น set RHOSTS
setg <OPTION> <VALUE>ตั้งค่าแบบ global ใช้ข้ามทุก module ใน session ปัจจุบัน
unset <OPTION>ลบค่าที่ตั้งไว้
run / exploitสั่งรัน module ที่เลือกไว้
backออกจาก module ปัจจุบัน กลับไปหน้าหลัก
sessions -lแสดง session ที่เปิดอยู่ทั้งหมด
sessions -i <ID>เข้าไปใน session ที่เลือก
jobs -lแสดง background job ที่กำลังรัน
03 OPTIONS ที่พบบ่อย
OPTIONความหมายตัวอย่าง
RHOSTS IP เป้าหมาย (Remote Host) รองรับ range / CIDR ด้วย set RHOSTS 10.10.10.5
set RHOSTS 10.10.10.0/24
RPORT พอร์ตเป้าหมาย (Remote Port) set RPORT 445
LHOST IP ของเครื่องเรา ที่ payload จะเชื่อมต่อกลับมา (สำคัญมากสำหรับ reverse shell) set LHOST 10.10.10.100
LPORT พอร์ตของเครื่องเราที่รอรับการเชื่อมต่อกลับ set
