OSINT · PASSIVE RECONNAISSANCE · KALI LINUX

theHarvester

คู่มือใช้งานเครื่องมือรวบรวมข้อมูลสาธารณะ (email, subdomain, ชื่อพนักงาน) ที่เกี่ยวกับโดเมนเป้าหมาย — ขั้นตอนแรกของงาน pentest ก่อนแม้แต่ nmap

เวอร์ชันอ้างอิง 4.4.0 แพลตฟอร์ม Kali Linux โดเมนตัวอย่าง example.lab (lab จำลอง)
⚠ คำเตือนสำคัญก่อนใช้งาน

theHarvester ใช้รวบรวมข้อมูลสาธารณะที่เกี่ยวกับ "โดเมน/องค์กร" เป้าหมาย เท่านั้น การใช้งานที่ถูกต้องคือ:

  • ✓ ตรวจสอบโดเมน/องค์กรของตัวเอง เพื่อดูว่ารั่วข้อมูลอะไรสู่สาธารณะบ้าง ก่อนที่ผู้โจมตีจะใช้ทำ phishing/social engineering
  • ✓ งาน pentest ที่มีสัญญา/scope of work ระบุโดเมนเป้าหมายชัดเจน

✗ ไม่ใช่ เครื่องมือสำหรับสะกดรอยหรือค้นหาข้อมูลส่วนตัวของบุคคลใดบุคคลหนึ่งโดยเฉพาะ การนำไปใช้ติดตาม คุกคาม หรือละเมิดความเป็นส่วนตัวของผู้อื่นผิดทั้งกฎหมายและจริยธรรม

ตัวอย่างทั้งหมดในคู่มือนี้ใช้โดเมนสมมติของแล็บทดสอบส่วนตัว example.lab เท่านั้น

01

theHarvester คืออะไร

theHarvester เป็นเครื่องมือ "passive OSINT" คือไม่ได้ส่ง packet ไปแตะเป้าหมายตรงๆ แบบ nmap แต่ใช้ดึงข้อมูลที่ "มีอยู่แล้วในที่สาธารณะ" จาก search engine และฐานข้อมูลออนไลน์ต่างๆ มารวบรวมไว้ในที่เดียว ขั้นตอนแรกของ pentest มักเริ่มจากเครื่องมือนี้ก่อน nmap ด้วยซ้ำ เรียกว่าขั้น Reconnaissance (Recon)

ข้อมูลที่รวบรวมได้

  • Email address ที่เกี่ยวข้องกับโดเมน เช่น john@company.com
  • Subdomain ของโดเมนเป้าหมาย
  • ชื่อพนักงาน (employee names) ที่หลุดจาก LinkedIn / public profile
  • IP address ที่เกี่ยวข้องกับโดเมน
  • URL ที่ index อยู่ใน search engine
  • ข้อมูลจาก Shodan — banner ของอุปกรณ์ที่เชื่อมเน็ตสาธารณะ

แหล่งข้อมูล (sources) ที่ดึงมา

Search Engine
Google, Bing, DuckDuckGo, Yahoo
Certificate Transparency
crtsh — ใบรับรอง SSL ที่ออกให้โดเมน
Code Repository
GitHub — ค้นหา credential/config ที่หลุด
Social
LinkedIn — ชื่อพนักงาน
Specialized (ต้อง API Key)
Shodan, Hunter.io, VirusTotal, SecurityTrails

การติดตั้งใน Kali

bash
# ปกติมีอยู่แล้วในชุด Kali
sudo apt update && sudo apt install theharvester -y

# ตรวจสอบเวอร์ชัน
theHarvester -h
02

Syntax พื้นฐาน

syntax
theHarvester -d <domain> -b <source> [OPTIONS]

รูปแบบทั่วไป

examples
theHarvester -d example.lab -b google
theHarvester -d example.lab -b all
theHarvester -d example.lab -b crtsh -f output.html
03

คำอธิบาย Flag ทั้งหมด

Flag คำอธิบาย
-d, --domain โดเมนเป้าหมาย (จำเป็นเสมอ) เช่น -d example.lab
-b, --source แหล่งข้อมูลที่จะดึง (จำเป็นเสมอ) เลือกได้ตัวเดียวหรือ all เพื่อดึงทุกแหล่งที่ไม่ต้องใช้ API key — ตัวเลือกที่ใช้บ่อย: google, bing, duckduckgo, yahoo, crtsh, github-code, linkedin, shodan, hunter, virustotal
-l, --limit จำกัดจำนวนผลลัพธ์ที่ดึงต่อแหล่งข้อมูล เช่น -l 200
-S, --start เริ่มดึงผลลัพธ์จากลำดับที่ N (pagination เก็บผลลัพธ์ต่อจากครั้งก่อน)
-f, --filename บันทึกผลลัพธ์ลงไฟล์ รองรับ .html, .json, .xml เช่น -f result.html
-c, --dns-brute เปิดการ brute-force subdomain แบบ DNS เพิ่มเติม (คล้าย gobuster โหมด dns แต่ฝังมาในตัว)
-n, --dns-resolve resolve subdomain ที่เจอให้ได้ IP address ด้วย ไม่ใช่แค่ชื่อ
-t, --dns-tld ค้นหา TLD อื่นๆ ของชื่อองค์กรเดียวกัน เช่น company.com, company.net, company.co.th
-v, --virtual-host ตรวจสอบ virtual host ของ IP ที่เจอ (resolve ย้อนกลับว่า IP นี้มีโดเมนอื่นแอบอยู่ด้วยหรือไม่)
-e, --dns-server ระบุ DNS server ที่ใช้ resolve เช่น -e 8.8.8.8
-r, --take-over เช็คว่า subdomain ที่เจอเสี่ยงต่อ "subdomain takeover" หรือไม่ — กรณี DNS record ชี้ไปยัง service ที่ถูกลบไปแล้ว เช่น S3 bucket/Heroku app ที่ไม่มีอยู่แล้ว
